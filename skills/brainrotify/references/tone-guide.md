# Brainrotify Tone Guide

Keep the joke readable. The goal is novelty, not semantic drift.

## Intensity

| Level | Target feel | Guidance |
| --- | --- | --- |
| `L` | playful | Add a few phrases like `lowkey`, `no cap`, `this slaps`, but keep the sentence mostly normal. |
| `Mid` | meme-heavy | Use more substitutions and louder sentence endings such as `we vibing` or `fr fr`. |
| `W` | aggressively online | Allow phrases like `chat`, `absolute cinema`, `we are so back`, but keep instructions decipherable. |

## Lexicon

Prefer short phrases that still preserve the original meaning:

- `important` -> `lowkey important`
- `note` -> `vibe check`
- `warning` -> `big yikes`
- `tip` -> `pro gamer move`
- `optional` -> `dealer's choice`
- `works` -> `slaps`
- `broken` -> `cooked`
- `fixed` -> `no longer cooked`
- `verify` -> `vibe check`

## Guardrails

- Preserve exact code, paths, commands, flags, URLs, and inline code spans.
- Avoid slurs, explicit sexual language, and anything that would make the text unusable in a normal engineering context.
- Do not change severity if the source text is a warning, requirement, or constraint.
- Keep headings and lists recognizable so the rewritten file is still easy to scan.
